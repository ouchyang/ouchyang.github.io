# IP · MAC · 子网：集群网络的寻址基石

在 AI 集群中，数千块 GPU 之间高效通信的前提是 —— **每个节点都能被精确找到**。
无论是 InfiniBand 的 LID/GID 还是以太网的 IP/MAC，所有网络通信的根基都是 **寻址（Addressing）**。

本文从最基础的三个概念出发，为理解 Scale-Out 集群网络打下地基。

## 2. MAC 地址：网卡的"身份证号"

### 2.1 什么是 MAC 地址？

**MAC (Media Access Control) 地址** 是网络接口控制器（网卡/NIC）的物理地址，
在设备出厂时由厂商烧录，具有 **全球唯一性**。

- **长度**: 48 位（6 字节）
- **表示方法**: 以十六进制表示，如 `08:00:27:4a:5c:e3`
- **唯一性**: 一张网卡对应一个唯一 MAC 地址；一台设备若有多张网卡，每张都有独立 MAC

### 2.2 MAC 地址的结构

| 部分 | 字节 | 分配者 | 示例 |
|------|------|--------|------|
| **OUI (组织唯一标识)** | 前 3 字节 | IEEE 统一分配给厂商 | `08:00:27` = Oracle/VirtualBox |
| **NIC (设备编号)** | 后 3 字节 | 厂商自行编号 | `4a:5c:e3` |

### 2.3 特殊 MAC 地址

| 地址 | 含义 |
|------|------|
| `ff:ff:ff:ff:ff:ff` | **广播地址** — 包被发送到本网段所有设备 |
| `01:xx:xx:xx:xx:xx` | **组播地址** — 最高字节最低位为 1 |

## 3. IP 地址：网络的"逻辑门牌号"

### 3.1 为什么有了 MAC 还需要 IP？

| 特性 | MAC 地址 | IP 地址 |
|------|---------|---------|
| 层次 | 扁平（无结构） | 层次化（网络号 + 主机号） |
| 分配依据 | 设备制造商 | **网络拓扑位置** |
| 可变性 | 出厂固定 | 可按需分配（DHCP 动态 / 静态配置） |
| 路由能力 | ✗ 仅限本地网段 | ✓ 支持跨网段路由 |
| 协议层 | L2 数据链路层 | L3 网络层 |

> **类比：** MAC 地址如同身份证号（一生不变，但不反映你住在哪里），IP 地址如同门牌号（标识了你在哪个小区、哪栋楼、哪层，搬家了地址就变）。

### 3.2 IPv4 地址格式

IPv4 地址为 **32 位（4 字节）**，使用 **点分十进制** 表示：

| 格式 | 值 |
|------|---|
| 二进制 | `11000000.10101000.00000001.01100100` |
| 十进制 | `192.168.1.100` |
| 每段 | 8 位 · 8 位 · 8 位 · 8 位 |

### 3.3 特殊 IP 地址

| 地址 | 含义 |
|------|------|
| `255.255.255.255` | 受限广播（仅本网段） |
| `0.0.0.0` | 本机 / 默认路由 |
| `127.0.0.1` | 本地回环（loopback） |
| `10.x.x.x` | 私有地址 (A 类) |
| `172.16–31.x.x` | 私有地址 (B 类) |
| `192.168.x.x` | 私有地址 (C 类) |

## 4. 子网与子网掩码：灵活划分网络

### 4.1 什么是子网？

**子网（Subnet）** 是将一个大的 IP 网络划分为多个较小的逻辑网段。通过子网划分可以：

- 缩小广播域，减少网络噪声
- 增强安全性（不同子网可设置访问控制）
- 提高 IP 地址利用率

> **CIDR 表示法：** `192.168.1.100/24` 表示前 24 位是网络号，等价于掩码 `255.255.255.0`。每个子网的第一个地址是 **网络地址**，最后一个是 **广播地址**，因此可用主机数 = 2^(主机位) - 2。

## 5. ARP：IP 到 MAC 的桥梁

### 5.1 为什么需要 ARP？

交换机基于 MAC 地址转发数据帧，但上层应用只知道目标的 IP 地址。
**ARP (Address Resolution Protocol)** 协议将 IP 地址解析为对应的 MAC 地址。

## 6. 路由：跨子网通信

### 6.1 关键原则

当源主机与目标主机不在同一子网时，数据包必须通过 **路由器** 转发。核心规则：

> **IP 地址在传输中始终不变，MAC 地址每经过一跳路由器都会改变。**

## 7. 三张表的协同工作

理解网络通信的关键在于理解 **路由表 · ARP 表 · MAC 表** 三者的协同配合。

逐步点击下面的动画，观察一个跨子网数据包从发出到送达经过的每一步查表过程：

<ThreeTableFlow />

| 表 | 维护者 | 映射关系 | 协议层 |
|----|--------|---------|--------|
| **路由表** | 路由器 / 主机 | 目的网段 → 下一跳 IP + 出接口 | L3 |
| **ARP 表** | 每台主机 | IP 地址 → MAC 地址 | L2/L3 |
| **MAC 表（转发表）** | 交换机 | MAC 地址 → 物理端口 | L2 |

## 8. 在 AI 集群网络中的意义

在万卡 GPU 集群中，这些基础概念无处不在：

- **子网划分**: 将数千个计算节点按 Rack（机柜）划分子网，是 Fat-tree 拓扑设计的基础
- **ARP 风暴**: 大规模集群中 ARP 广播可能成为性能瓶颈，因此 InfiniBand 使用 Subnet Manager 集中管理寻址
- **RDMA 寻址**: RoCE v2 基于 IP/UDP 封装 RDMA 协议，IP 地址和路由配置直接影响 NCCL 通信性能
- **ECMP 路由**: 等价多路径路由利用多条路径并行转发，依赖路由表的精确配置

理解了 IP、MAC 和子网，就掌握了 Scale-Out 集群网络的第一块拼图。
