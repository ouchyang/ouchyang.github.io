# IP · MAC · Subnet: Addressing Foundations for Cluster Networking

In AI clusters, efficient communication among thousands of GPUs starts with one prerequisite — **every node must be precisely addressable**. Whether it's InfiniBand's LID/GID or Ethernet's IP/MAC, all network communication is rooted in **addressing**.

This article covers the three foundational concepts for understanding Scale-Out cluster networking.

## 2. MAC Address: The NIC's "Identity Card"

### 2.1 What is a MAC Address?

A **MAC (Media Access Control) address** is the physical address of a network interface controller (NIC), burned in by the manufacturer at production time. It is **globally unique**.

- **Length**: 48 bits (6 bytes)
- **Notation**: Hexadecimal, e.g. `08:00:27:4a:5c:e3`
- **Uniqueness**: One NIC, one unique MAC; a device with multiple NICs has multiple MACs

### 2.2 MAC Address Structure

| Part | Bytes | Assigned By | Example |
|------|-------|-------------|---------|
| **OUI (Vendor ID)** | First 3 | IEEE, globally assigned | `08:00:27` = Oracle/VirtualBox |
| **NIC (Device ID)** | Last 3 | Vendor assigned | `4a:5c:e3` |

### 2.3 Special MAC Addresses

| Address | Meaning |
|---------|---------|
| `ff:ff:ff:ff:ff:ff` | **Broadcast** — sent to all devices in the subnet |
| `01:xx:xx:xx:xx:xx` | **Multicast** — lowest bit of first byte is 1 |

## 3. IP Address: The Network's "Logical Street Address"

### 3.1 Why Do We Need IP When We Already Have MAC?

| Property | MAC Address | IP Address |
|----------|-----------|-----------|
| Structure | Flat (no hierarchy) | Hierarchical (network + host) |
| Based on | Device manufacturer | **Network topology** |
| Mutability | Fixed at factory | Assignable (DHCP / static) |
| Routing | ✗ Local segment only | ✓ Cross-subnet routing |
| Layer | L2 Data Link | L3 Network |

> **Analogy:** MAC is like a national ID number (permanent, but doesn't tell where you live). IP is like a street address (reflects your location, changes when you move).

### 3.2 IPv4 Address Format

IPv4 addresses are **32 bits (4 bytes)** in **dotted decimal** notation:

| Format | Value |
|--------|-------|
| Binary | `11000000.10101000.00000001.01100100` |
| Decimal | `192.168.1.100` |
| Per octet | 8 bits · 8 bits · 8 bits · 8 bits |

### 3.3 Special IP Addresses

| Address | Meaning |
|---------|---------|
| `255.255.255.255` | Limited broadcast (local subnet only) |
| `0.0.0.0` | This host / default route |
| `127.0.0.1` | Loopback |
| `10.x.x.x`, `172.16–31.x.x`, `192.168.x.x` | Private addresses |

## 4. Subnets and Subnet Masks

### 4.1 What is a Subnet?

A **subnet** divides a large IP network into smaller logical segments to:

- Reduce broadcast domain size
- Enhance security with access control between subnets
- Improve IP address utilization

> **CIDR notation:** `192.168.1.100/24` means 24 bits for the network portion. Usable hosts per subnet = 2^(host bits) − 2 (network address + broadcast address).

## 5. ARP: Bridging IP to MAC

### 5.1 Why Do We Need ARP?

Switches forward frames by MAC address, but applications only know the destination IP. **ARP (Address Resolution Protocol)** resolves an IP address to its corresponding MAC address.

## 6. Routing: Cross-Subnet Communication

### 6.1 Key Principle

When source and destination are in different subnets, a **router** forwards the packet. The core rule:

> **IP addresses stay constant across hops; MAC addresses change at every hop.**

## 8. Relevance to AI Cluster Networking

- **Subnetting**: Organizing thousands of compute nodes by rack into subnets is fundamental to Fat-tree topology design
- **ARP Storms**: Large-scale ARP broadcasts can bottleneck clusters — InfiniBand uses a centralized Subnet Manager instead
- **RDMA Addressing**: RoCE v2 encapsulates RDMA over IP/UDP; IP and routing config directly affects NCCL communication performance
- **ECMP Routing**: Equal-Cost Multi-Path routing depends on precise routing table configuration for parallel forwarding

Understanding IP, MAC, and subnets is the first piece of the Scale-Out cluster networking puzzle.
